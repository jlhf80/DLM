# DLM Tutorial — Standalone Tutorial Design Spec

**Date:** 2026-04-29
**Status:** Approved — ready for implementation planning
**Scope:** Approach B (problem-first), 10 topic notebooks + 2 capstones
**Delivery:** Jupyter notebook series in a new GitHub repo `DLM-Tutorial`
**Target reader:** Masters-level industrial engineer with Python fluency, some probability/statistics background, no prior Bayesian state space modeling experience
**Goal:** Reader can correctly specify, fit, interpret, and compare DLM models at beginner and intermediate levels by the end of the series

---

## 1. Overview

A self-contained notebook tutorial that teaches Dynamic Linear Models through applied marketing problems — demand forecasting, seasonality, consumer behavior, advertising effectiveness, and pricing. Each notebook opens with a business question, motivates the DLM concept needed to answer it, derives the relevant equations with explicit W&H and Petris citations, implements the model using `statsmodels` and `PyMC`, and closes with exercises and a worked mini project.

**Primary references:**
- West, M. & Harrison, J. — *Bayesian Forecasting and Dynamic Models* (2nd ed.) — cited as **W&H**
- Petris, G., Petrone, S. & Campagnoli, P. — *Dynamic Linear Models with R* — cited as **Petris**

**Package philosophy:** fully independent of the DLM project engine. All filtering, smoothing, and estimation go through `statsmodels.tsa.UnobservedComponents` (classical) and `PyMC >= 5.0` (Bayesian). No shared code with the `DLM` repo.

## 4. Notebook anatomy

Each of notebooks 01–09 follows this six-section problem-first template.

### 4.1 Marketing scenario (2–3 cells)
Opens with a concrete business question. A simulated dataset is generated inline using `numpy.random.default_rng(42)`, saved to `data/NN_<name>.csv`. Simulation parameters are chosen to make the relevant DLM behavior clearly visible.

### 4.2 Intuition (2–3 cells)
Prose and a plot showing why a naive approach (rolling mean, OLS, simple exponential smoothing) fails or misses something important. Bridges the marketing problem to why a state space model is the right tool.

### 4.3 Theory and derivation (3–5 cells)
The DLM equations for the chapter's model type, derived step by step in LaTeX. Explicit W&H and Petris section citations at each major result. Notation follows W&H throughout.

### 4.4 Implementation (3–5 cells)
1. Fit using `statsmodels.tsa.UnobservedComponents`
2. Where applicable (notebooks 05–09), re-fit or extend using PyMC
3. Visualise filtered state mean + credible intervals, one-step-ahead forecasts, residuals
4. Interpret outputs in marketing terms

### 4.5 Exercises (3–4 cells)
`# YOUR CODE HERE` cells with assert-based auto-checks where feasible. At least one exercise per chapter requires changing the model specification, not just tuning a parameter. Solutions in `solutions/NN_<name>_solutions.ipynb`.

### 4.6 Chapter mini project (4–6 cells)
A fully worked extension of the chapter concept to a slightly more complex marketing scenario. No blanks — reader runs and reads. Distinguishes from exercises by being narrative and complete.

### Capstones (P1, P2)
Template: business brief → EDA → model selection rationale → full fit → forecast and business recommendation. Fully narrated, no blanks.

- **P1 — Demand forecasting:** weekly CPG product demand with trend, seasonality, and promotional lift. 12-week forecast with uncertainty intervals.
- **P2 — Marketing attribution:** paid search + display advertising channels plus price, each with time-varying coefficients. Compares dynamic regression DLM to a static marketing mix model. Concludes with channel attribution and ROI estimate.

## 5. Toolchain and dependencies

### 5.1 Core packages

| Purpose | Package |
|---|---|
| Kalman filtering / smoothing / MLE | `statsmodels` (`UnobservedComponents`) |
| Bayesian inference | `pymc >= 5.0` |
| Trace plots and MCMC diagnostics | `arviz >= 0.17` |
| Numerics | `numpy`, `scipy` |
| Plotting | `matplotlib`, `seaborn` |
| Data handling | `pandas` |
| Baseline comparison | `scikit-learn` |
| Notebook smoke tests | `nbmake` |

`scikit-learn` is used in notebooks 07 and 08 for baseline comparison models (Ridge regression, simple exponential smoothing pipeline) to give readers a familiar anchor before DLM residual diagnostics and Bayes factors.

### 5.2 pyproject.toml structure


### 5.3 Simulated data
Each notebook generates its own dataset at the top, seeds `numpy.random.default_rng(42)`, and saves to `data/NN_<name>.csv`. No external datasets; no internet required. All simulations use marketing-realistic parameter ranges.

## 7. Non-goals

- No custom Kalman filter engine — all filtering via statsmodels or PyMC
- No real datasets — all simulated
- No R code or Quarto output
- No advanced topics (FFBS, interventions, multivariate) — those belong to the Advanced series in the DLM repo
- No GPU/JAX backend
- No integration with the DLM repo codebase

## 8. Approach A outline (saved for later)

A textbook-linear alternative. Same target reader, same packages, same 10-12 notebook count. Sequencing follows W&H chapter order: math foundations first, model types second, estimation third, Bayesian fourth, capstones last. Marketing framing is lighter — problems appear as examples within chapters, not as the opening hook.

**Proposed chapter order:**
1. Introduction to State Space Models — representation, notation, W&H §2
2. The Kalman Filter — derivation, forward pass, W&H §4
3. Smoothing and Forecasting — RTS smoother, k-step-ahead, W&H §4.5
4. Specifying DLMs — local level, local linear trend, superposition, W&H §3
5. Seasonal Models — factor form and Fourier form, W&H §7
6. Dynamic Regression — time-varying coefficients, W&H §9
7. Parameter Estimation — MLE via statsmodels, discount factors, W&H §6
8. Bayesian DLM — conjugate updates, PyMC inference, W&H §4.4, §15
9. Model Diagnostics — residuals, Bayes factors, AIC/BIC, W&H §4.5, §10
10. Interventions and Structural Breaks — outliers, monitoring, W&H §11
11–12. Capstones — same P1/P2 as Approach B
