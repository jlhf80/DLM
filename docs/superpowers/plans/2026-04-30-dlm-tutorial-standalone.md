**Cell 3 [markdown]:**
```
## What is a DLM?

A Dynamic Linear Model (DLM) is a state space model of the form:

**Observation equation:**  y_t = F_t' θ_t + v_t,   v_t ~ N(0, V_t)

**Evolution equation:**  θ_t = G_t θ_{t-1} + w_t,  w_t ~ N(0, W_t)

The *state* θ_t is unobserved. The Kalman filter computes the posterior
distribution of θ_t given observations y_1, ..., y_t (W&H §4.2).
```

**Cell 4 [code] — quick statsmodels demo:**
```python
import numpy as np
from statsmodels.tsa.statespace.structural import UnobservedComponents
import matplotlib.pyplot as plt

rng = np.random.default_rng(0)
T = 52
y = np.cumsum(rng.normal(0, 2, T)) + rng.normal(0, 5, T) + 100

model = UnobservedComponents(y, level='local level')
result = model.fit(disp=False)

fig, ax = plt.subplots(figsize=(10, 4))
ax.plot(y, alpha=0.5, label='Observed')
ax.plot(result.filtered_state[0], label='Filtered level')
ax.legend()

plt.tight_layout()
plt.show()
print("statsmodels OK")
```

**Cell 4 [markdown] — theory:**
```
## Local Level Model (W&H §2.3, §4.2; Petris §2.1–2.2)

The **local level model** (random walk plus noise) is the simplest DLM:

**Observation:** y_t = μ_t + v_t,   v_t ~ N(0, V)

**Evolution:**   μ_t = μ_{t-1} + w_t,  w_t ~ N(0, W)

Here μ_t is the unobserved "true level". V controls observation noise;
W controls how fast the level can drift.

The Kalman filter recursion (W&H §4.2) gives the posterior:

  (μ_t | y_{1:t}) ~ N(m_t, C_t)

where m_t = m_{t-1} + K_t (y_t − m_{t-1}),  K_t = R_t / Q_t

and R_t = C_{t-1} + W,  Q_t = R_t + V (W&H Eq. 4.7–4.8).
```

**Cell 4 [markdown] — theory:**
```
## Local Linear Trend (W&H §3.1, §4.3; Petris §2.3)

Adds a stochastic slope μ_{t}:

  y_t = μ_t + v_t,               v_t ~ N(0, V)
  μ_t = μ_{t-1} + β_{t-1} + w_μ,  w_μ ~ N(0, W_μ)
  β_t = β_{t-1} + w_β,            w_β ~ N(0, W_β)

The slope β_t evolves as a random walk. The filter now tracks
two states: (μ_t, β_t). When W_β is small, the slope changes
slowly — ideal for detecting gradual trend reversals (W&H §4.3).
```

```python
rng = np.random.default_rng(42)
T = 156
sigma_obs, sigma_level = 50.0, 3.0
period = 52

level_true = np.zeros(T)
level_true[0] = 500.0
for t in range(1, T):
    level_true[t] = level_true[t-1] + rng.normal(0, sigma_level)

y = level_true + seasonal_true + rng.normal(0, sigma_obs, T)
```

**Cell 3 [markdown] — theory: MLE:**
```
## Maximum Likelihood Estimation (W&H §6.1–6.3; Petris §4.1)

The Kalman filter computes the log-likelihood as a by-product:

  log p(y_{1:T}) = Σ_t [ -½ log(2π Q_t) - e_t² / (2 Q_t) ]

where e_t = y_t - f_t is the one-step forecast error and Q_t its
variance (W&H §4.4, Eq. 4.13). Maximising over (V, W) gives the
MLE. statsmodels does this automatically via `model.fit()`.
```

**Cell 4 [code] — MLE fit and likelihood surface:**

**Cell 5 [markdown] — discount factor:**
```
## Discount Factors (W&H §6.3)

Instead of estimating W explicitly, a **discount factor** δ ∈ (0,1)
inflates the prior variance at each step:

  R_t = C_{t-1} / δ

Typical values: δ=0.9 (moderate adaptation) to δ=0.98 (slow adaptation).
This avoids ML optimisation and is robust when T is small (W&H §6.3).
```

**Cell 6 [code] — discount filter implementation:**
```python
def local_level_discount(y, delta, V, m0=None, C0=None):
    T = len(y)
    m = np.zeros(T); C = np.zeros(T)
    m[0] = y[0] if m0 is None else m0
    C[0] = V if C0 is None else C0
    for t in range(1, T):
        R = C[t-1] / delta
        Q = R + V
        K = R / Q
        m[t] = m[t-1] + K * (y[t] - m[t-1])
        C[t] = (1 - K) * R
    return m, C

fig, axes = plt.subplots(1, 3, figsize=(15, 4), sharey=True)
for ax, delta in zip(axes, [0.80, 0.90, 0.98]):
    m_disc, _ = local_level_discount(y, delta, V=V_hat)
    ax.plot(y, alpha=0.3); ax.plot(level_true, color='black', lw=1.5)
    ax.plot(m_disc, color='tomato', lw=2)
    ax.set_title(f'δ={delta}'); ax.set_xlabel('Week')
axes[0].set_ylabel('Units')
plt.suptitle('Discount factor: smaller δ → more responsive filter')
plt.tight_layout(); plt.show()
```

```
## Bayesian DLM (W&H §4.4; Petris §4.4–4.5)

Instead of point-estimating V and W, the fully Bayesian approach

  V ~ π(V),  W ~ π(W)

PyMC's GaussianRandomWalk represents the state θ_{1:T} directly
```


**Cell 3 [markdown] — theory:**
```
## Model Comparison (W&H §10.8; Petris §4.2)

Given two models M_1 and M_2, the **log Bayes factor** is:

  log BF_{12} = log p(y | M_1) - log p(y | M_2)

where log p(y | M_i) is the log marginal likelihood. Under equal
priors, BF > 1 (log BF > 0) favours M_1.

When marginal likelihoods are unavailable, **BIC** approximates:

  log BF_{12} ≈ -(BIC_1 - BIC_2) / 2     (W&H §10.8)

AIC penalises complexity less than BIC — better for prediction;
BIC is more consistent for model selection.
```

**Cell 4 [code] — fit four competing models:**
```python
model_specs = {
    'Local level': dict(level='local level'),
    'Local linear trend': dict(level='local linear trend'),
    'Level + seasonal(h=1)': dict(level='local level',
                                   freq_seasonal=[{'period': 52, 'harmonics': 1}]),
    'Level + seasonal(h=2)': dict(level='local level',
                                   freq_seasonal=[{'period': 52, 'harmonics': 2}]),
}

results_cmp = {}

    results_cmp[name] = UnobservedComponents(y, **spec).fit(disp=False)

comparison = pd.DataFrame({
    name: {'loglik': r.llf, 'AIC': r.aic, 'BIC': r.bic, 'nparams': r.df_model}
    for name, r in results_cmp.items()
}).T.sort_values('BIC')

print(comparison.to_string(float_format='{:.1f}'.format))
```

**Cell 5 [code] — log Bayes factor via BIC:**
```python
def log_bf_bic(r1, r2):
    """Approx log Bayes factor: -(BIC1 - BIC2)/2."""
    return -(r1.bic - r2.bic) / 2

best = list(results_cmp.values())[comparison.index.get_loc('Level + seasonal(h=2)')]
for name, res in results_cmp.items():
    lbf = log_bf_bic(best, res)
    print(f"log BF(seasonal h=2 vs {name}): {lbf:.2f}")
```
